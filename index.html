Four-Year Automated Course Planner

GitHub Repo: automatic-course-planner

Project Description

Our project is an automatic course planner tailored for first-year Olin students. It is a mix of interactive visualization that outputs a personalized four-year course schedule when the user inputs their major, whether or not they will take an LOA, graduate early, study abroad, and which senior capstone they will take. Information about how to use our course planner is in the next section. Our full Github repository is here: https://github.com/olincollege/automatic-course-planner.

How to Use Our Course Planner

To use our course planner, start by cloning our Github repository (by running "git clone https://github.com/github_username/repo_name.git") and install the requirements by running "pip install -r requirements.txt". Next, run the course_planner_main.py file and respond to the prompts that come up on the window to generate your four-year course schedule! For more detailed information, visit the README on our Github!

Interactivity

Our project is used by the user by first inputting their constraints: major, whether or not they will take an LOA, whether or not they will study abroad, whether or not they will graduate early, and what senior capstone they will take. Our system then uses a predictive algorithm that uses past course offerings to guess what courses will be offered in the next years. Taking into account user constraints and major requirements, a table of eight (or less) semesters of courses will be shown. If the user likes the schedule shown, they have the option to save the file to their computer as a CSV file. If not, the user can prompt the program to generate another schedule. Eventually add a video/screenshot of our program being used.

Architecture

  • Model: returns a dataframe with full schedule based on user input constraints (major, study abroad, LOA, graduating early) and predicted course offerings.
  • View: shows tkinter window, dropdown, and button
  • Controller: handles user input and updates our model with different constraints
